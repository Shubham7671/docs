### 1. Request Your API Key

<Note>
  To use the QuickReel API, you'll need an API key. Get yours from the
  [QuickReel Dashboard](https://api.quickreel.io/).
</Note>

### 2. Base URL

All API requests should be made to:

```bash
https://engine.quickreel.io/api/v2
```

### 3. Authentication

<CodeGroup>
```bash cURL
curl --request POST \
  --url https://engine.quickreel.io/api/v2/clip \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'Content-Type: application/json'
```

```javascript Node.js
const headers = {
  "x-api-key": "YOUR_API_KEY",
  "Content-Type": "application/json",
};
```

```python Python
headers = {
    'x-api-key': 'YOUR_API_KEY',
    'Content-Type': 'application/json'
}
```

</CodeGroup>

## API Response Format

All API responses follow a consistent JSON format:

```json
{
  "statusCode": 200,
  "status": "success",
  "message": "Operation completed successfully",
  "data": {
    // Response data specific to the endpoint
  }
}
```

## Rate Limits and Quotas

<Info>
  Our API implements rate limiting to ensure fair usage and optimal performance
  for all users.
</Info>

- **Free Tier**: 100 requests/hour
- **Pro Tier**: 1000 requests/hour
- **Enterprise**: Custom limits available

### Example Error Response

<CodeGroup>
```json 400 Bad Request
{
  "statusCode": 400,
  "status": "error",
  "message": "Invalid or missing required field: videoUrl",
  "errors": [
    {
      "field": "videoUrl",
      "message": "Must be a valid URL"
    }
  ]
}
```

```json 401 Unauthorized
{
  "statusCode": 401,
  "status": "error",
  "message": "Invalid API key provided"
}
```

```json 429 Rate Limit
{
  "statusCode": 429,
  "status": "error",
  "message": "Rate limit exceeded. Please try again in 60 seconds",
  "retryAfter": 60
}
```

</CodeGroup>
